# i18n 翻译风格指南

## 通用原则

1. **忠于原意,不直译**:传达概念,不必字字对应
2. **保持技术准确性**:技术术语按术语库译,代码示例不译
3. **语气一致**:与英文版保持一致的礼貌级别
4. **简洁**:避免冗余的敬语填充
5. **一致**:同一术语在全文档统一

## 不翻译的内容

- 品牌名:Airbyte、Stripe、Postgres、Databricks、Snowflake、BigQuery 等
- 代码示例、命令行、文件路径、URL
- 配置字段名(JSON key)
- 环境变量名
- 错误码、API 名称
- 引用原文的方括号标记如 `[required]`

## 翻译策略

### 代码块
```bash
# 原文注释不译
poetry install
```

### 占位符

Docusaurus i18n 使用 ICU MessageFormat:

```json
{
  "connector.count": "已发现 {count, plural, one {# 个连接器} other {# 个连接器}}",
  "sync.status": "同步 {streamName} 状态:{status}"
}
```

### 链接

- 内部链接保持英文路径 `/integrations/sources/postgres`
- 锚点保持原样
- 外链保持原 URL

### 图片与媒体

- 截图文字暂保留英文(后续视情况增加多语言字幕)
- alt 文字必须翻译
- Markdown 图片 `![alt](path)` 中的 `alt` 翻译

## 各语言补充说明

### 简体中文 (zh-Hans)

- 使用全角中文标点:`,。、;:`?!"`
- 第一人称用"我们",与英文 "we" 对应
- 段落标题使用阿拉伯数字编号,不用中文数字
- 数字与英文混排时,中文在前加空格:`共 100 个 connector`

### 日语 (ja)

- 敬体 `です・ます` 与英文对应
- カタカナ外来语保留原文
- 句号用 `。`,引号用 `「」`

### 巴西葡萄牙语 (pt-BR)

- 葡萄牙语(巴西)用 `tu` 或 `você`,避免过于正式
- 日期格式:DD/MM/YYYY
- 货币:R$ (BRL)

## 审查清单

翻译完成后自检:

- [ ] 术语与术语库一致
- [ ] 占位符 `{var}` 完整保留
- [ ] 内部链接未破坏
- [ ] Markdown 语法仍正确
- [ ] 代码块未翻译
- [ ] 不译的内容保持原样
- [ ] 段落标题层级正确
- [ ] 没有英文残留(术语除外)
